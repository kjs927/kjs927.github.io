Building CI/CD Pipelines with IDEA, Gitlab, Jenkins and Kubernetes - Topic 2.2 - Jenkins (Pull, Build and Deploy)

Freestyle and Maven Project tutorial

  1. Install Maven
    (1). sudo apt-get install maven

    (2). sudo vim /etc/profile

    (3). source /etc/profile

    (4). mvn -v

    (5). sudo vim /usr/share/maven/conf/settings.xml

    (6). Configure Maven on Jenkins

    • Global Tool Configuration

      add JDK


      add Maven
    • Configure System


      fill “Environment variables” form, apple and save

    (7). Test Maven

    • select the “ssh-item” (we did before)

    • select Configuration
    • find Build
    • add Execute shell, apply and save
    • click Build now
    • check output
    • done!

  2. Install Tomcat
    (1). sudo apt-get install tomcat9

    (2). the default port is 9000 (NAT by 29000 here)

    (3). we need an admin account

    (4). sudo vi /etc/tomcat9/tomcat-users.xml

    (5). sudo vi /etc/tomcat9/server.xml option

    (6). sudo systemctl enable tomcat9
    (7). sudo systemctl restart tomcat9
    (8). done!

  3. Test and deploy an application
    (1) Freestyle project

    • create it
    • get git url

    • pull test


    • set up build settings


    • build test

    • install deploy plugin


    • configure deployment settings



    • add credential for tomcat

    • test deploy

    • done!

    (2). Maven project

    • install Maven Integration


    • create maven project


    • set up build settings
    • configure deployment settings

    • test it

    • error caused by jenkins jdk
    • add correct jdk
    • configure project jdk

    • build it again
    • done!
