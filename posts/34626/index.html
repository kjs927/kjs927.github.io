Building CI/CD Pipelines with IDEA, Gitlab, Jenkins and Kubernetes - Topic 2.1 (Jenkins and Gitlab)

The leading open source automation server, Jenkins provides hundreds of plugins to support building, deploying and automating any project. - jenkins.io

  1. Install
    1.1 install Jenkins

    • (1) sudo apt-get install openjdk-8-jdk

    • (2). wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo apt-key add -

    • (3).

    • (4). sudo apt-get update

    • (5). sudo apt-get install jenkins

    • (6). sudo vim /etc/default/jenkins

    • (7). sudo vim /etc/init.d/jenkins

    • (8). sudo systemctl enable jenkins

    • (9). sudo systemctl start jenkins

    • (10). first login

    • (11). sudo cat /var/lib/jenkins/secrets/initialAdminPassword

    • (12). copy and paste the password

    • (13). select “Install suggested plugins”

    • (14). create first admin user

    • (15). set jenkins url

    • (16). jenkins is ready

    1.2 install “Role-based Authorization Strategy” plugin

    • (1). login
    • (2). Manage Jenkins -> Manage Plugins

    • (3). select “Available” tab and search “Role-based Authorization Strategy”
    • (4). mark it and click “Download now and install after restart”


    • (5). login
    • (6). Manage Jenkins -> Configure Global Security
    • (7). change “Authorization Strategy” to “Role-Base Strategy” and save
    • (8). go back to “Manage Jenkins”
    • (9). find “Manage and Assign Roles”

    • (10). select “Manage Roles”
    • (11). add base role

    • (12). add item roles (for projects)
    • (13). apply and save
    • (14). go back to “Manage Jenkins”
    • (15). select “Manage Users”
    • (16). create test users (charlie and david)


    • (17). login with charlie

      oops
    • (18). login with admin bob
    • (19). assign role to charlie, Manage Jenkins->”Manage and Assign Roles”

    • (20). login with charlie again (OK)
    • (21). login with bob again, assign roles to charlie and david
    • (22). create projects for testing roles





    • (23). charlie’s view
    • (24). david’s view

  2. Add credentials
    2.1 by username and password

    • (1). Manage Jenkins->Manage Credentials

    • (2). click “global”
    • (3). add credential

    • (4). back to dashboard
    • (5). select a project
    • (6). fill git form


    • (7). save, back to project and “Build Now”
    • (8). click “Build History” item
    • (9). check “Console Output”, done!

    2.2 by ssh

    • (1). ssh-keygen -t rsa
    • (2). cd /home/webdev/.ssh/ && ll id_rsa and id_rsa.pub are important!
    • (3). add ssh key on Gitlab
      • login Gitlab
      • add id_rsa.pub content to settings -> SSH Keys


    • (4). add ssh key on Jenkins
      • enter “Manage Credentials”

      • add ssh credential by the id_rsa content

    • (5). back to dashboard
    • (6). “New Item”
    • (7). get ssh url from Gitlab project
    • (8). fill git form on Jenkins, apply and save
    • (9). “Build Now”
    • (10). check the result, done!
