What's the Difference Between CI and CD (and the Other CD)?

  • Abstract from RedHat [What is CI/CD?]

    • Continuous integration (CI) helps developers merge their code changes back to a shared branch, or “trunk,” more frequently—sometimes even daily. If automated testing discovers a conflict between new and existing code, CI makes it easier to fix those bugs quickly and often.
    • Continuous delivery (CD), the goal of continuous delivery is to have a codebase that is always ready for deployment to a production environment.
    • Continuous deployment (CD), as an extension of continuous delivery, continuous deployment means that a developer’s change to a cloud application could go live within minutes of writing it (assuming it passes automated testing). This makes it much easier to continuously receive and incorporate user feedback.
